Help/ User Guide

Background:

This shiny App is developed to allow analysis of Road Safety Digital Breath Test Data from England and Wales for year 2014. This Shiny App allows dynamic selection of data columns/features from the target dataset. Selected data columns/features will be dynamically summarized in the backend server and result visually presented back at the front end dashboard.

User Interface

Different areas of the Shiny App:

Description/ Purposes

Ability to switch between:

  1. Welcome Note – A welcome message and basic information.
  2. Dashboard - The Main App section :-
    1. Visual Plot Viewing Area (Interactive Data Presentation in Visual Mode)
    2. Data Columns/Features Selection Area (Y-axis, X-axis Data Column Selection)
    3. Dataset Grouping Options (Grouping of records)
    4. Dataset Summary Area (Visual Plot’s Dynamic Dataset Summary, Complete Digital Breath Test Raw Dataset Summary)
    5. Visual Plot Selection Area (Bar, Line or Pie Chart viewing)

    1. Background information about the App.

  4. Help/User Guide – this page on simple user guide and FAQ.

 

 

FAQ/ How to…

Filtering on the Legend

Q:

There’s too many trend lines, bars etc… on the Graph/VIsual Plot. The Graph/Visual Plot is a mess, I can’t see clearly the data that I am interested in.

A:

Place mouse cursor on the Graph’s Legend section. Click on each desired legend item to activate filter. Click again to deactivate the filtering on the specific legend item.

*NOTE: The legend filtering functionality is not available for Pie Chart Visual Plot

 

Zooming View into the Dataset

Q:

How can I zoom in to review onto a smaller section of the Visual Plot/Graph?

A:

Option 1: Place mouse cursor on top right corner of Graph, shows the hidden Plotly function options. Use the “zoom” or “zoom in” functionality.

 

Option 2: Place mouse cursor on Graph area of interest. Click and drag a window selection area to zoom in.

 

Using the Data Table Browser

Q:

How can I make use of the Data Table Browser?

A:

You can either make use of the Generic Search String input box (top right corner of the Table Browser Area), or

 

Use the filtering input box on top of each columns to do record searching/filtering.

 

Data Table Browser showing all record rows from the Raw Dataset

Q:

Why does the Data Table Browser shows a lot more record rows than what is seems depicted by the Visual Plots/Graph?

A:

The Data Table Browser works independently from the Visual Plots. It is meant to shows all record rows from the raw dataset. It purpose is to facilitate easier data lookup, data searching, data comparison and etc…

 

The Visual Plots are produced by summarizing and grouping data columns/features from the entire raw dataset. Data used for generating the Visual Plots are hence just a subset of the complete Raw Dataset.

 

X-axis selection are specially filtered depending on Visual Plot Type Selection

Q:

Why does the x-axis option differs when the plot type is switched between Bar, Line or Pie chart?

A:

Certain data column/features are only suitable for specific plot type, e.g. : Line chart is more suited for time series related column/features and more useful for showing trend changes over time hence the x-axis  selection are filtered to best suit the type of Visual Plot chosen.

 

“Group by” options are specially filtered depending on Visual Plot Type Selection

Q:

Why does the “group by” option differs when the plot type is switched between Bar, Line or Pie chart?

A:

Certain data column/features are only suitable for specific plot type, e.g. : Pie chart usually make use of only 2 dimension data. Adding a third dimension data to Pie chart may not necessary make sense. Hence for Pie chart, “group by” option is disabled and x-axis is used as the categorization/separation feature.
